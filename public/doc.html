Introduction

onpostt is a decentralized protocol built on a relay/server-based architecture, supporting both HTTP and WebSocket communication. It features digitally signed and encrypted blocks—each block is signed with a private key, verified with a public key, and optionally encrypted for secure transmission. Designed to be censorship-resistant, onpostt distributes data across multiple relays and servers, making it ideal for social networks, messaging platforms, and any application where decentralization and freedom are critical.

Key Features

  • Node.js execution environment
  • Uses PostgreSQL instead of JSON files
  • Improved performance and scalability with relational database
  • Organized queries with custom filters and parameters
  • Highly scalable architecture
  • WebSocket and P2P communication

Security and Privacy

With private key-based encryption, users have full control over their data. Decentralization ensures censorship resistance and eliminates single points of failure.

Scalability and Flexibility

The transition to a PostgreSQL-based architecture provides greater scalability, performance, and flexibility—enabling seamless integration with other technologies while ensuring efficient data management.

cd onpostt

// Start the server

// HTTP and WebSocket server started on port 3000

Quick Start

Here's a basic example to get started with onpostt:

// Define servers for HTTP and WebSocket connections
const servers = {
  host: ['http://localhost:3000'],
  socket: ['ws://localhost:3000']
};

// Connect to the defined servers
const connect = onpostt.connect(servers);

// Async function to create, sign, and send a post block
async function post() {
  // Private key used for signing the block
  const privateKey = '21e28dfffa49daf6373527c579ee16dea1afd7c8a2f95d9eb2e6aeb0a8d6d3d2';

  // Generate the public key from the private key
  const pubkey = onpostt.generatePublicKey(privateKey);

  // Create the block object with post details
  const block = {
    pubkey: pubkey,
    created_at: Math.floor(Date.now() / 1000), // Timestamp in seconds
    mode: "post",
    query: [
      ["username", "lanpião.dev"],
      ["site", "google.com"]
    ],
    content: 'Rei do Cangaço',
    app: 'mariabonita.com.br'
  };

  // Sign the block with the private key
  const BlockSigned = await onpostt.signBlock(block, privateKey);

  // Send the signed block to the relays
  onpostt.sendBlock(BlockSigned, function(response) {
    console.log('Relay response:', response);
  });
}

Architecture

Decentralized Network

Data is stored in decentralized blocks, signed and encrypted with the user's private key. No central authority controls the network.

Relay Transmission

Uses a relay-based approach for data transmission between nodes, ensuring efficient propagation of information across the network.

Field Type Description
pubkey String Public key of the user
created_at Number Unix timestamp of block creation
mode String Type of event (e.g., "post")
query Array Key-value pairs for filtering
content String Main content of the block
app String Application that created the block
id String Unique identifier of the block
sig String Digital signature of the block

API Reference

connect()

Establishes connection to one or more relay servers.

// Connect to multiple relays
onpostt.connect(['ws://relay1.example.com', 'ws://relay2.example.com']);

Note

The connection will automatically failover to other relays if one goes down.

signBlock()

Signs a block with the user's private key.

// Sign a block
let signedBlock = await onpostt.signBlock(block, privateKey);
Parameter Type Description
block Object The block object to sign
privateKey String User's private key (hex format)

sendBlock()

Sends a signed block to the connected relays.

// Send a block
onpostt.sendBlock(signedBlock, function(response) {
  console.log('Relay response:', response);
});

sub()

Subscribes to blocks matching specific criteria.

// Subscribe to posts from a specific user
onpostt.sub({
  mode: 'post',
  query: [["username", "lanpião.dev"]],
  limit: 10
}, function(event) {
  console.log('New event:', event);
});
Parameter Type Description
filters Object Filter criteria for subscription
callback Function Function to call when matching events arrive

generatePublicKey()

Generates a public key from a private key.

// Generate public key
let pubkey = onpostt.generatePublicKey(privateKey);

Examples

Basic Usage

// 1. Connect to relays
onpostt.connect(['ws://localhost:3000', 'ws://localhost:4000']);

// 2. Create and sign a block
async function createPost() {
  const privateKey = 'your-private-key-here';
  const pubkey = onpostt.generatePublicKey(privateKey);

  const block = {
    pubkey: pubkey,
    created_at: Math.floor(Date.now() / 1000),
    mode: "post",
    query: [["username", "myusername"]],
    content: 'Hello onpostt network!',
    app: 'myapp.com'
  };

  const signedBlock = await onpostt.signBlock(block, privateKey);
  onpostt.sendBlock(signedBlock, function(response) {
    console.log('Block sent!', response);
  });
}

// 3. Subscribe to posts
onpostt.sub({
  mode: 'post',
  query: [["username", "myusername"]],
  limit: 10
}, function(event) {
  console.log('New post:', event);
});

Advanced Queries

// Subscribe with multiple query parameters
onpostt.sub({
  pubkey: '03dca175856ff79a1eb5d3b368b6840af29c38c36bf3291d07573ddcdf59110523',
  mode: 'post',
  query: [
    ["username", "lanpião.dev"],
    ["site", "mariabonita.com.br"]
  ],
  app: "mariabonita.com.br",
  limit: 10,
  offset: 0
}, function(handleEvent) {
  console.log('Filtered events:', handleEvent);
});

Tip

You can combine multiple filters to precisely target the content you want to receive. The more specific your filters, the less data needs to be transferred.
